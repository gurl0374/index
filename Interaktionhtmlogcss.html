Html, CSS og Grid

Html

For at kode skal man bruge en editor. Jeg har valgt at bruge Brackets, men der findes mange andre som man kan bruge. Det handler bare om, at finde en der passer til dig. Html er den del, hvor du skriver alle de oplysninger du ønsker i dit website. Grov sagt indeholder Html´en , en head og en body og en footer (hoved, krop og fod. Aller øverst oppe angiver, du at det er en html fil. I html filen øverst oppe angiver, du hvilket sprog du ønsker at bruge. I head skriver man også meta tags. De angiver hvilket tegnsæt man vil bruge, og om siden skal være responsiv m.v. I head skriver du også titlen på din website. Det er den som er siden navn. Der kan man også lave søgeord til søgemaskiner, så din side kan søges frem. I body koder du alt, hvad din side skal inderholder. Footeren er afslutningen på din side.

Til toppen

Begreber i html

Her er forklaret, nogle af de begreber man kommer ud for i html. Tags eller elementer:Tags starter med mindre end tegnet og slutter med større end tegn. Det der kommer efter er dit indhold. Det sluttes af med et mindre end tegn en /dit tagnavn igen og så sluttes det af med en større end tegn. Tagnavnet kan være h1, som betyder heading altså overskrift. Der findes flere størrelser og h1 er den største. De andre hedder h2,h3,h4 o.s.v. Dette er blot en af mange tag navne. Her ved siden af er vist forskellige tags med indhold.

Flere begreber

Sematiske tags er beregnet til at kunne læses af både menesker og maskiner. h1(heading) er foreksempel et semantisk tag, fordi at man vil kunne læse teksten i mellem tagene.

Ikke Sematiske tags: kan ikke umildbart læses af både mennesker og maskiner. Span er for eksempel et ikke semantisk tag.

Validering: bør man gøre tit. Jeg bruger W3C. Man tjekker at der ikke er nogle alvorlige fejl. Erros. Den fortæller også hvor fejlen er, så det er lettere at finde. Den finder fejlene hvis der for eksempel mangler slut tags eller hvis der er for mange.

Struktur: Der bør være struktur i dit html eller CSS dokument. Det er lettere at overskue, hvis der er lave indryk i stedet for i en lang række.

Farverkoder i html og CSS: Farver kan difineres på flere måder. Enten ved at skrive farvens navn men man kan også bruge farvekoder F.eks. Hvid hedder #FFFFFF i HEX kode, RGB(255,255,255) kode eller white.

CSS

CSS er den del, hvor du styler alle de oplysninger du har i dit html. Man kan godt style i html. Jeg har valgt, at bruge strong, hvor jeg synes, det skulle være en mere markant skrift. Men set tilbage ville det nok havde været mere smart at have stylet det i CSS. Når man styler i CSS, skal der bruges selectors til at styles. Man kan style flere tags afgangen eller bare en tag ad gangen. Det du styler sætter du ind i en deklaration. Den starter med en krøllet parantes, så skriver man det der skal påvirkes, derefter kolon og til sidst hvad det skal påvirkes med. Slut af med semikolon.

(kilde: powerpoint om CSS :Niels Østergaard )

Begreber i CSS

Syntaks. Der bør være struktur i dit html eller CSS dokument. Det er lettere at overskue, hvis der er lave indryk i stedet for i en lang række.

Selectors

Selectors Selector bliver brugt til at finde eller vælge html elementer ud fra deres element navn, id, class,attributer m.v.

Selector typer

  • Element selectors også kaldt Type selectors

  • Class selectors

  • ID selectors

  • Atribute selectors

  • Pseudo-class selectors

  • Pseudo-elements

Af disse typer bruges 3 af dem rigtig meget.

Element selector bruges til alle elementer med et specifik paragraf tag.  F,eks. tags som p, h1, body.  Når du bruger denne selector, bliver alle paragraf tags med det specifike navn påvirket af den styling du bruger. Man kan også style flere tags ad gangen. Så skal man bare huske, at skrive et komma i mellem de forskellige tags. Eksempel: h1 { font-family: san-sarif;}

Når flere elementer skal have ens style, Den bruges når tags er mærket op med en klasse. F.eks. .class { CSS declaration;}

Den sidste der bliver brugt meget er ID selector.Bruges når der kun er ét element der skal styles. Man giver sit HTML-tag et id som er unikt for den. Hvis man kommer til at navn give flere med samme ID kan der godt komme kluder i det. CSS sætter man er# foran sit ID og styler den ellers som normalt.

Ellers kan nævnes 2b typer struktuelle selectors. Descentant selector som er flere type selectorer, hvor den ene er efterkommer af den anden, eks: nav a. Kræver at a ligger inden i nav

Child selectors: Fungerer næsten som en descendant, men child er første efterkommer (altså barn, og ikke barnebarn) Parent: en hvilken som helst gyldig selektor Child: En selektor som filtrerer child elementerne

Grid

Nå du styler i CSS er det en stor hjælp at bruge Grid. Grid er system hvor dine felter afgøre hvor bredden af dine tekst og billeder er. Et Grid er et usynligt gitter, der opdeler siden i forskellige sektioner, og det kan understøtte både vertikale og horisontale kolonner. Det gør siden mere overskuelig, og den fremstår ikke rodet. Grids er altså en form for strukturering af en hjemmeside.

Størrelsen på gridfeltene kan man kode må flere måder. I billedet ovenfor, er der brugt %, px og auto.
Dette er en måde man kan kode det på i CSS.
.container
{
grid-template-columns: 40px 50px auto 50px 40px; grid-template-rows: 25% 100px auto;
}

Her er en anden måde at opbygge det på.

.item-a
{
grid-column-start: 2;
grid-column-end: five;
grid-row-start: row1-start
grid-row-end: 3
}
